classmethod sort_hidden_nodes(conn_mat, latest_possible=True)[source]¶

Topologically sort hidden nodes given a connectivity matrix.

This is essentially a scheduling problem. A node can only be be computed once the source nodes of incoming edges have been computed. Additionally, it is preferable to have a minimum number of layers. Many schedules can potentially fullfil these requirements. Two notable once are earliest possible scheduling and latest possible scheduling. For the numpy implementation the sorting makes no difference (as long the number of layers does not change). For the torch implementation it is preferable to compute nodes at the latest possible moment to reduce the required memory in earlier layers. Hence, this is what we will use as a default.

Parameters

  • conn_mat (np.ndarray) – Connection matrix of the hidden nodes.

  • latest_possible (bool, optional) – If true (default), latest possible scheduling is used, else earliest possible scheduling.
